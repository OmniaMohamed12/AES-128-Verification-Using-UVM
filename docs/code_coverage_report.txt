    20                                               integer i;
    21                                               /*
    22                                               	We generate all the keys needed in the encryption and decryption at the beginning of the encryption or decryption
    23                                                and store them, then we use them in the AES_Encrypt and AES_Decrypt modules as needed according to the current 
    24                                                round.
    25                                               */
    26                                               /*
    27                                                The functions:
    28                                                1) subwordx() applies a table lookup to all to all four bytes of the sent word. subwordx() calls the function
    29                                               	 c() four times, each time it sends to c() 1-byte to perform the table lookup on it.
    30                                                2) rconx() contains the values given by [x^(i-1),{00},{00},{00}], with x^(i-1) being powers 
    31                                               	 of x (x is denoted as {02}) in the field GF(28).
    32                                                3) rotword() applies a cyclic shift of the bytes in a word. For example,{09cf4f3c} is changed into {cf4f3c09}
    33                                               	 after applying this function.
    34                                               */
    35                                               
    36                                               /*
    37                                               		The pseudo-code of the this algorithm is found in the NIST.pdf attached to the repository with some modification
    38                                               	in the code to fit with verilog.
    39                                               */
    40                                               /*
    41                                               		For simplicity, We are going to explain the storing mechanism of the generated keys on an example of
    42                                               	128-bit key.It would be easy to apply the same concept on 192-bit and 256-bit keys. We would explain it in one 
    43                                               	round only. The next rounds would perform the same operations.
    44                                               	
    45                                               	-The example:
    46                                               		-Note that in case of 128-bit key w[0:1407].
    47                                               		
    48                                               		1) when w=key then w= {1279*{0}:key} where 1279*{0} means that the first 1279 bits are all zero valued
    49                                               			and the end of the w array contains the current 128-bit key.
    50                                               		2) when temp = w[(128 * (nr + 1) - 32) +: 32] then temp=w[1376 +:32] so temp in the first round would contain
    51                                               			the last 32-bit word of the the current key. 
    52                                               		3) After performing (temp = SubWord(RotWord(temp)) xor Rcon[i/Nk]) or (temp = SubWord(temp)), We would perform
    53                                               			(new1 = w[(128*(nr+1)-(nk*32))+:32] ^ temp) which is (new1=w[1280+:32] ^ temp) in the first round,
    54                                               			where w[(128*(nr+1)-(nk*32))+:32] here is equivelent to w[i-Nk] in the pseudo-code. Now we have the new1
    55                                               			generated key word (new1) and we need to add it at the end of (W) array.
    56                                               		4) We would shift W by 32-bit to the left to empty space to the new1 generated key word.
    57                                               		5) w = {w[0 : (128 * (nr + 1) - 32) - 1], new1} is w={w[0:1375],new1} where w now contains w={1247*{0}:key:new1} 
    58                                               			where 1247*{0} means that the first 1247 bits are all zero valued and they are followed by the original 
    59                                               			128-bit key, which are followed by the new1 generated 32-bit key.
    60              1                        564     		6) Repeat this process for the rest of the rounds.At the end of all the rounds we would have all the W array
    61                                               			filled with all the keys.
    62              1                        564     	
    63              1                        564     */

    64              1                      22560     always@* begin
    65                                               //The first [(nk*32)-1 ]-bit key is stored in W.
    66              1                       5640     	w = key;    
    67              1                       5640     	for(i = nk; i < 4*(nr + 1); i = i + 1) begin
    68              1                       5640     	temp = w[(128 * (nr + 1) - 32) +: 32];
    69              1                       5640     	if(i % nk == 0) begin
    70                                               		rot = rotword(temp); // A call to the function rotword() is done and the returned value is stored in rot.
    71                                               		x = subwordx (rot);	//A call to the function subwordx() is done and the returned value is stored in x.
    72                                               		rconv = rconx (i/nk); //A call to the function rconx() is done and the returned value is stored in rconv.
    73                                               		temp = x ^ rconv;   
    74              1                      22560     	end
    75                                               	else if(nk >6 && i % nk == 4) begin
    76              1                      22560     		temp = subwordx(temp);
    77              1                      22560     	end
    78                                               	new1 = (w[(128*(nr+1)-(nk*32))+:32] ^ temp);
    79                                               	// We would shift W by 32 bit to the left to add the new1 generated key word (new1) at its end.
    80                                               	w = w << 32;
    81                                               	w = {w[0 : (128 * (nr + 1) - 32) - 1], new1};
    82                                               end
    83                                               
    84                                               end
